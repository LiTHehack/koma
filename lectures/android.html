Responsivitet

Progressive enhancement

Ganska välanvänt begrepp – kan betyda flera saker. Coolt sedan 2011. En lösning för att lösa problemet att skapa system för flera olika enheter.
Vi kommer fokusera på betydelsen att med så enkla medel som möjligt, göra något anpassat för så många enheter som möjligt.
Detta kommer vi kika på mer än PE, som snarare fokuserar på att ge användare med hög kapacitet mycket.
Problemet med responsivitet är att det kan bli segt för telefoner att över 3G ladda ned samma bild som över bb.

AndroidManifest.xml

Styr appen intern och externt.

Fil direkt i roten, alltså högst upp i hiearkin, måste finnas med rätt namn.
XML-formatterad, likt html.
Styr hur andra appar kan interagera med din app, men också grundläggande saker som hur systemet ska starta din app och vilka tillstånd som behövs.
Massa spännande och ospännande, och jättetekniskt, läs!

res/

Bilder, ljud, vyer, mm.

Resurser och tillgångar till appen.
Allting är ganska separerat i android, vyer är på ett annat ställe än logik.
Bryta ut koden så möjligt, skapa överskådliga strukturer och gör allt lättare att hantera. Fixa med en bild utan att härja med kod.

  • res/
    • drawable/
      • icon.png
      • selfie.jpg
      • bird.png
    • layout/
      • main.xml
      • info.xml
    • values/
      • strings.xml
      • styles.xml

Exempel på hiearki. Bilder hör hemma i drawable-mappen, vyer i layout och i values går det att spara saker som textsträngar, stiler osv
Inga undermappar, tänk på det.
Fler grejer, menu, raw, xml, animator
Går att anpassa beroende på telefonen och användarens inställningar.
